Propuesta de expansión

Nuevas dinámicas para First Impact

Catálogo de ideas concretas — torres, enemigos, reacciones químicas, eventos de mapa, modos y progresión meta — pensadas para entrelazarse con los sistemas que ya tenés en el código. Cada propuesta incluye cómo encaja en la arquitectura actual (specs, oleadas, mazo, reliquias) para que estimar el costo de implementación sea directo.

13 secciones 40+ propuestas 4 fases de roadmap

01 Lectura del juego actual Punto de partida

Antes de proponer cosas, recapitulo lo que ya existe — para que las propuestas se apoyen sobre estos sistemas en lugar de competir con ellos.

🏰
7 torres
core

Archer / Cannon / Sniper / Missile / Laser / Ice / Enhance. Data-driven vía TOWER_SPECS, escalan hasta nivel 25 (20 manual + 5 enhance).

👾
6 sub-tipos enemigo
core

Normal, Green (rápido), Blue (curador), Flying, Boss, Split y Bonus como flags combinables. Oleadas mixtas a partir de la 20.

🃏
Mazo + 11 reliquias
core

Mazo de cartas-torre, tienda entre oleadas, reliquias acumulables (Interest, Weaken, Bloodlust, Veteran…).

🌍
4 biomas
cosmético

Plain / Forest / Desert / Mountain. Hoy solo cambian colores de fondo y árboles — no afectan gameplay.

⚙️
Editor de mapas
core

Pintura libre con undo/redo (50 pasos), BFS para el path, guardado/carga en archivos .map.

💥
Sistemas de combate
core

Crits (base 10%), AoE, slow (Ice), enhance aura, formaciones de 3+, daño escalado por kills (Bloodlust).

Filosofía de las propuestas: casi todo se conecta con un sistema existente. Cuando una idea requiere infraestructura nueva, lo marco explícitamente con requiere refactor.

02 Nuevas torres 10 propuestas

Cada torre tiene una mecánica que la hace memorable — no solo "más daño con otros stats". Pensadas para encajar en TOWER_SPECS + un campo nuevo behavior_kind que reemplace los switch tower.type hard-coded de hoy.

⚡
Tesla
cadena

Rayo eléctrico que rebota entre hasta 4 enemigos cercanos. Daño decae 25% por rebote. Aplica estado SHOCK.

range 3.0 · dmg 4 · cd 1.2s · cost $65 sinergia brutal con enemigos WET (+100% daño)
🧪
Alquimista
DoT

Lanza viales de ácido. Aplica POISON: daño por segundo durante 5s, stackeable hasta 5 veces.

range 3.5 · dmg 2 + 1/s × 5s · cd 0.8s · cost $50 sinergia POISON + BURN = explosión
🌑
Pozo de gravedad
control

No dispara. Atrae a los enemigos hacia su centro cada 4s, retrasándolos ~1 celda en el path. Funciona con voladores.

range 3.5 · cd 4s · cost $80 sinergia agrupa enemigos para los AoE
💰
Banquero
economía

No dispara. Genera +$5 al final de cada oleada. Si está en formación con otro Banquero, +$10. Hasta nivel 10.

range 0 · cost $120 sinergia Interest Boost cuenta el income
🎯
Asesino
execute

Crit base 40%. Si un enemigo tiene <25% HP, garantiza el crit y lo ejecuta directo si el daño lo supera.

range 5.5 · dmg 12 · cd 1.8s · cost $75 sinergia Bloodlust amplifica los execute
📡

Solo dispara a voladores. Daño muy alto, ignora el cooldown de torres adyacentes — actúa como un "extra attack" del grupo.

Cómo encajan en el código

El motor ya soporta lo necesario con una pequeña extensión a Tower_Spec. Agregar un campo behavior_kind permite registrar comportamientos sin tocar el switch monolítico de update_towers: 

// Propuesta: extensión de Tower_Spec en constants.odin
Tower_Behavior :: enum {
    PROJECTILE,    // Archer, Cannon, Sniper, Missile, Asesino, Espía
    BEAM,          // Laser
    PULSE_AOE,     // Ice
    AURA,          // Enhance, Bardo
    PASSIVE,       // Banquero
    CHAIN,         // Tesla
    DOT,           // Alquimista
    LOBBED_AOE,    // Mortero
    PULL,          // Pozo de gravedad
    EXECUTE,       // Asesino (puede compartir PROJECTILE con flag)
    SPAWN_HELPER,  // Nigromante
    MARKER,        // Vidente
}

Tower_Spec :: struct {
    type:           Tower_Type,
    behavior:       Tower_Behavior,
    // ... campos existentes ...
    chain_targets:  i32,    // para CHAIN
    chain_falloff:  f32,    // damage falloff por rebote
    dot_duration:   f32,    // para DOT
    min_range:      f32,    // para LOBBED_AOE
    aura_speed_mul: f32,    // para AURA tipo Bardo
}

Con eso, update_towers pasa de su switch tower.type actual a un switch spec.behavior reutilizable. Agregar una torre nueva = una entrada en TOWER_SPECS + (a veces) un proc handler nuevo.

03 Nuevos enemigos 10 propuestas

El sistema actual usa flags booleanos (is_green, is_blue, etc.) — la propuesta es migrar a un Enemy_Spec tabla declarativa (ver sección 13), pero estas ideas funcionan en cualquiera de las dos formas.

🛡️
Centurión
escudo

Tiene un escudo que absorbe el primer 30% del daño. Si no recibe daño por 2s, el escudo se regenera 1%/s.

HP 1.6× · speed 0.7× · spawn wave 12+ contra torres de DoT (escudo no regenera)
🌀
Saltador
teletransporte

Cada 4s salta hacia adelante 3 celdas del path. Durante el salto es invulnerable. Aterriza brevemente vulnerable.

HP 0.6× · speed 1.0× · spawn wave 15+ contra Pozo de gravedad, Sniper bien posicionado
🥚
Necrótico
resurrección

Al morir, deja un huevo que rompe en 3s y revive un enemigo con 40% HP. Solo se revive una vez.

HP 1.0× · speed 0.9× · spawn wave 18+ contra el huevo es destruible — AoE lo elimina
📯
Comandante
buffer

Aura que da +20% velocidad y +15% HP a enemigos en rango de 2 celdas. Su prioridad debería ser "first" — mátenlo rápido.

HP 1.3× · speed 0.8× · spawn wave 25+ contra torres con alta prioridad de target
⛏️
Topo
stealth

Cada 3s se entierra por 2s — invulnerable y no puede ser objetivo. Mientras está bajo tierra ignora obstáculos.

HP 0.9× · speed 1.1× · spawn wave 20+ contra Ice tower lo expone (no se puede enterrar slowed)
👯
Espejo / Mímico
copia

Spawnea con stats genéricas. Al recibir daño por primera vez copia el HP del último enemigo muerto en el frame.

HP variable · speed 1.0× · spawn wave 30+ contra matar pequeños primero para que copie HP bajo
🩸
Berserker
enrage

A <50% HP gana +50% velocidad. A <25% gana +100% velocidad y +1 daño al goal si llega.

HP 1.2× · speed 0.9× · spawn wave 14+ contra ráfaga de daño alto (Sniper, Asesino), no DoT
👻
Espectro
resistencia

Inmune a daño físico (Archer/Cannon/Sniper/Missile). Solo afectado por Laser, Tesla, Alquimista e Ice.

HP 0.7× · speed 1.2× · spawn wave 22+ contra obliga a diversificar el arsenal
⛓️
Conductor
cadena

Comparte 30% del daño que recibe con enemigos cercanos. Bueno y malo — Tesla lo ama, AoE también, daño directo se desperdicia.

HP 0.9× · speed 0.9× · spawn wave 17+ contra los voladores no comparten
🐉
Boss multi-fase
jefe

A 66% HP gana shield. A 33% gana enrage. A 10% comienza a teletransportarse. Cada fase otorga oro al alcanzarla.

HP 18× · spawn wave 30, 60, 90 recompensa escalonada por fase
Pacing: introducir tipos nuevos cada ~5 oleadas y nunca dos sub-tipos nuevos en la misma oleada. El jugador necesita una "oleada de aprendizaje" antes de que el tipo se mezcle con otros.

04 Sistema de estados núcleo de las sinergias

Hoy el único estado es SLOW del Ice tower. Proponemos un sistema general de "status effects" con 6 estados y una matriz de reacciones — al estilo Divinity o Genshin Impact, pero adaptado a TD.

Los 6 estados base

❄️ Cold

El frío del Ice tower

Velocidad ×0.4 durante 2s. Existente. Reemplaza al "slow" actual.

💧 Wet

Mojado

Aplicado por Ice al deshelar, por lluvia, o por torres nuevas tipo "manguera".

🔥 Burn

Quemadura

DoT: 1.5 dmg/s durante 4s. Aplicado por Laser críticos o por Mortero (nuevo).

☠️ Poison

Veneno

DoT stackeable, ignora escudos. Aplicado por Alquimista (nueva torre).

⚡ Shock

Choque eléctrico

Stun de 0.3s. Aplicado por Tesla en críticos.

🎯 Mark

Marcado

+30% daño recibido durante 5s. Aplicado por Vidente o por reliquia "Cazador".

Matriz de reacciones

La gracia está acá. Combinar estados produce efectos secundarios — algunos brutales, otros que te perjudican si los activás mal.

Wet + Shock
⚡ Electrocución
Daño ×3 en el tick del shock. Combo más fácil y poderoso del juego.
Wet + Cold
🧊 Frozen
Stun completo 1.5s. Mientras esté frozen, próximo golpe físico = ×2 daño (Shatter).
Wet + Burn
💨 Steam
Anti-sinergia: cancela ambos estados y cura 5% HP al enemigo. Castigo a builds desordenados.
Burn + Poison
💥 Detonación
Explota: AoE pequeña con el daño acumulado restante de ambos DoT.
Cold + Burn
🌫️ Mist
Genera niebla en 1 celda durante 4s — torres en esa celda pierden 30% rango. Tactic-defensive: el enemigo lo causa al recibir mal el combo.
Mark + Shock
🎯⚡ Execute
Si HP < 30%, ejecución instantánea. Sinergia para Tesla + Vidente builds.

Encaje en el código

// Reemplazo de los campos slow_factor/slow_timer en Enemy
Status_Kind :: enum { COLD, WET, BURN, POISON, SHOCK, MARK }

Status_State :: struct {
    timer:     f32,         // 0 = no activo
    intensity: f32,         // usado por POISON stacks, COLD factor
    source:    Tower_Type,  // para tracking de daño/sinergias
}

Enemy :: struct {
    // ... campos existentes ...
    statuses: [Status_Kind]Status_State,
}

// Cada vez que se aplica un status nuevo, se chequea la matriz de reacciones
status_apply :: proc(e: ^Enemy, kind: Status_Kind, dur, intensity: f32) {
    // Check reaction with all currently active statuses
    for other_kind in Status_Kind {
        if e.statuses[other_kind].timer > 0 {
            trigger_reaction(e, kind, other_kind)
        }
    }
    e.statuses[kind] = Status_State{timer = dur, intensity = intensity}
}
Por qué este sistema cierra el círculo: hoy FROZEN_AMP (la reliquia) ya premia pegarle a enemigos lentos. Con este sistema, esa reliquia se vuelve la primera de toda una familia de reliquias temáticas (Pyromancer = +daño a burn, Toxin = +daño a poison, etc.).

05 Sinergias y combos depth multiplier

Las reglas que premian construir un sistema, no torres aisladas. Apunta a que cada partida tenga una "identidad" — la run del fuego, la del veneno, la del oro infinito.

Combo Meter

Matar enemigos en rápida sucesión carga un medidor visible arriba del HUD. Cuando se llena (~25 kills en menos de 10s), todas las torres entran en "Overdrive" por 5s: cooldown ×0.5, crit chance +30%. Castiga las pausas pero recompensa builds de ráfaga.

Sin combo meter

Cada kill vale lo mismo

Bloodlust suma micro-bonus por kill pero no hay incentivo a optimizar el "ritmo" de la oleada.

Con combo meter

El ritmo importa

Tower placements que dejan pasar primero a un tank para masacrar lo que viene atrás dejan de ser óptimos. El jugador piensa en "burst windows".

Set bonuses de reliquias

Tener 3+ reliquias de una "familia" desbloquea un bonus pasivo. Premia foco temático:

06 Biomas vivos y clima dinámico de cosmético a gameplay

Hoy los biomas solo cambian colores. Vale la pena: ya pagaste el costo de tenerlos definidos, lo único que falta son los modificadores. Y agregar un sistema de "clima" sobre el bioma multiplica la rejugabilidad sin arte nuevo.

Cada bioma trae reglas propias

BiomaModificador positivoModificador negativoSensación
🌿 Plain +10% oro por kill Sin penalización Bioma "default" — onboarding, runs estándar
🌲 Forest +15% daño físico −10% rango (terreno denso) Build melee — corta distancia, alto daño
🏜️ Desert +25% efectividad de Laser/Tesla Ice cooldown +30% Build elemental, energía sobre física
⛰️ Mountain +25% crit chance global Costo de upgrade +20% Run minimalista — pocas torres bien posicionadas

Clima dinámico

Cada 10 oleadas el clima cambia, anunciado con un toast 2 oleadas antes. Crea ventanas de oportunidad y obliga a improvisar.

🌧️
Lluvia
3-5 oleadas

Todos los enemigos en el path tienen estado Wet permanente. Tesla y Ice ganan poderío. Laser pierde 15%.

⛈️
Tormenta eléctrica
2-3 oleadas

Cada 5s un rayo cae sobre un enemigo random — 50 dmg garantizado. Tu propia infraestructura ayuda.

🌫️
Niebla
2-4 oleadas

Todas las torres ven su rango reducido en 20%. Los voladores son invisibles a más de 3 celdas.

☀️
Sequía
3-5 oleadas

Laser +50% pero Ice deshabilitado. Enemigos +10% velocidad.

❄️
Ventisca
3 oleadas

Todos los enemigos llegan ya en estado Cold (lentos). Pero el oro por kill se reduce 30%.

🌑
Eclipse
1 oleada — rare

Solo ocurre 1-2× por partida. Crit chance global ×3 pero todos los enemigos vienen como split-types.

Sub-sistema de pre-roll: ya tenés lookahead_bonus[3] para anunciar oleadas bonus. La misma idea sirve para anunciar el clima 2 oleadas antes — campo weather_next: Weather en Simulation.

Las celdas del mapa son hoy semánticamente pobres (EMPTY / PATH / SPAWN / GOAL / OBSTACLE / TOWER_*). Agregar tiles con reglas propias multiplica el espacio de diseño del editor.

Tiles nuevos para el editor

🪨

tile

Torres construidas acá ganan +15% rango. Visualmente, una piedra elevada.

💎
CRYSTAL_NODE
tile

Torres adyacentes ganan +15% daño. Hasta 1 torre puede aprovecharlo simultáneamente.

🌊
RIVER
tile

Path que aplica Wet a enemigos que lo cruzan. No bloquea voladores ni enemigos enterrados.

🔥

Aplica daño + Burn a quienes pisan. Las torres no se pueden construir aquí.

🌪️

🚫

Eventos en partida

Mid-run, cosas que pasan en el mapa y obligan a reaccionar. Aparecen acompañadas de un toast con countdown visual.

🌋

cada 15 olas

Una celda EMPTY random se vuelve LAVA durante 5 oleadas. Si tenías torre ahí, se destruye y refunda 50%.

🏗️

⭐

random

💰

🏛️

🚪

09 Economía y comercio expandir la tienda

Hoy la decisión económica es "¿upgradeo o construyo torre nueva?". Las siguientes mecánicas hacen el side-game económico más interesante sin volverlo abrumador.

📈
Mercado fluctuante
tienda

Cada visita a la tienda, una rareza random tiene -25% precio. Otra tiene +25%. Premia esperar el momento de comprar.

🎲
Apuesta de oleada
pre-wave

Antes de iniciar la oleada, podés apostar $X a "mato todo sin perder vidas". Si lo lográs, $X × 2. Si no, perdés el oro.

🏦
Préstamo
avanzado

Pedí hasta $300 adelantados. Devolver en 5 oleadas o pagar 50% de interés extra por cada oleada pasada.

🛒
Mercader errante
rare

Aparece una vez por partida (entre olas 20-40). Ofrece 1 reliquia UNIQUE con descuento del 50%, solo esa partida.

♻️
Reciclado
venta

Vender una reliquia activa devuelve 60% del precio × stacks acumulados. Permite re-planificar mid-run.

🔄
Reroll mejorado
tienda

Hoy un solo reroll cuesta $50. Propuesta: rerolls progresivos $25 / $50 / $100 — incentiva pensar antes que rerolear.

Cuidado con desbalanceos: el préstamo y la apuesta son herramientas de high-risk high-reward. Limitalas (1 préstamo activo a la vez, máximo 3 apuestas por partida) para que no se vuelvan obligatorias.

Bestiario / Codex

Cada enemigo, torre y reliquia visto se desbloquea en un "Codex" accesible desde el menú principal. Lleva estadísticas: cuántos kills, mejor ola alcanzada, % de partidas con esa reliquia. Trivia opcional para cumplir con el feel sci-fi (ficha del enemigo, lore corto).

Logros

~30 logros que desbloquean cartas iniciales, mapas y modos:

  • Pacifista — Completá la ola 20 sin construir Snipers.
  • Glass cannon — Ganaste la partida con menos de 5 torres.
  • Frio absoluto — Mantené 50+ enemigos en estado Cold simultáneamente.
  • Rey del dinero — Terminá una oleada con $1000+ en el banco.
  • Solo de un instrumento — Ganá usando solo un tipo de torre.

13 Roadmap priorizado orden sugerido

Casi todo lo de arriba depende de un refactor previo. Acá ordeno qué hacer en qué fase para que cada paso desbloquée los siguientes sin tirar trabajo a la basura.

Fase 0 — Refactor previo ~3-5 días

Antes de agregar contenido nuevo, conviene pagar la deuda técnica que el code review ya identificó. Sin esto, cada propuesta de abajo cuesta el doble.

  • Tabla declarativa RELIC_SPECS + relic_stacks: [Card_Kind]i32 en Simulation.
  • Migrar enemigos a Enemy_Spec con campos data-driven.
  • Agregar behavior_kind a Tower_Spec + split de update_towers en procs por behavior.
  • Arreglar el flujo Menu → Play (bug pendiente del review).
Fase 1 — Contenido base nuevo ~1-2 semanas

Aprovechar el refactor para meter contenido que ya encaja en los slots existentes.

  • 3-4 torres nuevas (Tesla, Alquimista, Asesino, Banquero) — encajan en TOWER_SPECS con los behaviors nuevos.
  • 3-4 enemigos nuevos (Centurión, Saltador, Necrótico, Berserker) — encajan en ENEMY_SPECS.
  • 6-8 reliquias nuevas — encajan en RELIC_SPECS.
  • Activar los modificadores de bioma — costo: ~30 minutos por bioma.
Fase 2 — Sistemas profundos ~2-3 semanas

Sistemas que multiplican el valor del contenido existente.

  • Sistema de status effects (sección 4) con matriz de 6 reacciones.
  • Combo Meter como HUD persistente.
  • Set bonuses de reliquias (necesita que la fase 1 ya haya añadido las reliquias temáticas).
  • Clima dinámico — 4-5 climas iniciales.
  • 3-4 tipos de wave nuevos (Sprint, Iron March, Elite, Stealth).
Fase 3 — Meta y replay ~3-4 semanas

Cosas que dan razón para volver a jugar el día después.

  • Sistema de estrellas + árbol de talentos persistente (necesita un archivo de save aparte).
  • Ascension levels.
  • Codex / bestiario.
  • Logros (30+).
  • Modos Endless y Daily Run.
Fase 4 — Expansión ~1 mes+

Las propuestas más ambiciosas — se justifican una vez que el juego ya tiene una comunidad.

  • Tiles especiales (River, Lava, Portal, Crystal Node) + editor support.
  • Eventos de mapa mid-run.
  • Modos Puzzle, Hardcore, Versus asincrónico.
  • Resto de torres (Bardo, Nigromante, Mortero, Vidente, Pozo de gravedad, Espía).
  • Boss multi-fase y resto de enemigos.

Heurísticas de balance

  • Una mecánica nueva por release. Si soltás status effects y combo meter el mismo día, los jugadores no van a entender qué les pegó.
  • Counters explícitos. Cada enemigo nuevo debe tener al menos 2 torres del arsenal actual que lo contrarresten claramente, para que el jugador descubra la solución sin tutorial.
  • Tope de stacks. Hoy las reliquias no tienen tope — con 20 stacks de Bloodlust el juego se rompe. Considerar caps suaves con curva decreciente.
  • Test las anti-sinergias. Wet+Burn = Steam que cura es divertido pero también frustrante — playtest con jugadores nuevos antes de soltarlo.
